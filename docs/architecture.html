3D沉浸式实时语音交互视频

这份文档面向项目维护和展示讲解：说明当前系统如何把裸眼 3D、场景编排、实时语音对话、AI 回应与最终展示页连接成一个互动叙事体验。

目录 项目定位 页面入口 技术栈 文件地图 后端服务 前端运行模式 场景与数据 互动管线 运行部署

项目定位

3D沉浸式实时语音交互视频 是一个基于 Canvas 的三维互动视频原型。它不把视频当作单向播放内容，而是把图层、声音、用户视角、语音输入和剧情状态组合成可参与的叙事系统。

运行方式 Node.js 原生 HTTP 服务
默认地址 127.0.0.1:5174
前端形态 Editor / Viewer / Final / Intro
核心素材 PNG、GIF、MOV、WebM、音频

Spatial

裸眼 3D 视差

通过摄像头或移动端姿态输入获取用户视角，再按图层深度调整投影位移，形成无需 3D 眼镜的空间感。

Narrative

动态剧情分支

每个场景都可配置结束方式：停留、自动进入下一场景，或等待用户对话并按关键词跳转。

Voice

AI 角色回应

用户语音或文字会进入 Kimi 导演提示，再由讯飞 TTS 合成为角色语音，回到前端播放与字幕展示。

页面入口

项目现在由多个 HTML 入口组成，但主要交互逻辑集中在同一个前端文件中，通过页面上的 body[data-mode] 区分运行模式。

入口 定位 说明
intro.html 项目总览 面向展示与介绍，包含英雄视频、项目说明、演示视频、移动端 Demo、创作思路和技术问答。
intro-demo.html 移动端快速体验 复用渲染系统，隐藏编辑控件，以嵌入方式播放指定场景，并优先使用移动设备姿态输入。
index.html 场景编辑器 导入素材、调整图层、配置场景组、设置剧情流转、测试语音对话和 TTS。
viewer.html 只读演示页 用于播放单个场景，保留基础场景选择和启动控制。
final.html 最终展示页 全屏体验，包含摄像头视角追踪、场景组卡片、播放控制、语音/文字交互和新手说明弹窗。

技术栈

后端

  • Node.js 原生 http 服务
  • 原生 fs、zlib、crypto
  • 静态文件、Range 请求、Gzip 响应
  • Kimi 与讯飞 TTS 服务端代理

前端

  • 原生 HTML / CSS / JavaScript
  • Canvas 2D 透视投影
  • DOM 视频、GIF 与音频叠加播放
  • Web Speech API 语音识别

视觉与体验

  • 深绿黑背景、薄荷绿强调色、暖金辅助色
  • 8px 圆角的工具面板与按钮
  • 全屏沉浸式舞台
  • 项目总览页使用真实视频素材承载首屏

后端服务

server.js 是单文件 Node 服务，默认监听 127.0.0.1:5174。它同时负责页面访问、素材上传、场景数据读写和外部 AI/TTS 代理。

接口 方法 用途
/health GET 服务健康检查；返回数据库状态，PM2 / 反向代理探活使用。
/api/layout GET / POST / DELETE 读取、保存、删除场景布局；支持列表与详情读取。
/api/settings GET / POST 保存场景组、当前编辑组和最终展示组等全局设置。
/api/assets GET / POST 列出素材库，或接收编辑器上传的新素材。
/api/director-cue POST 把用户输入、剧情阶段和上下文发送给 Kimi，生成角色回应与舞台提示。
/api/tts POST 调用讯飞超拟人 TTS，返回完整音频 Buffer。
/api/tts-stream POST 讯飞 TTS 流式推送；以分块响应逐步将音频数据发回前端，降低首包延迟。
/api/script-beats GET 返回剧情节拍配置（opening / choice / reveal / ending），供编辑器填充导演提示。
/api/log-error POST 接收前端上报的运行时错误，写入服务端结构化日志。

前端运行模式

src/app.js 通过 editor、viewer、final、intro-demo 四类模式控制界面能力。渲染、素材管理、场景加载、语音和 TTS 管线尽量复用同一套状态与函数。

Editor 模式

提供完整编辑工具，包括素材导入、图层参数、场景组管理、流程配置、语音对话验证和最终展示组选择。

Viewer 模式

用于查看单个场景，界面更轻，适合排查场景播放和视差表现。

Final 模式

面向最终体验，默认暂停等待用户确认，包含摄像头模块、场景组选择、播放按钮、语音和文字输入。

Intro Demo 模式

作为 intro 页面中的手机端体验模块，隐藏编辑控件，聚焦移动端姿态驱动的三维视差演示。

场景与数据

场景数据存放在 data/scene-layout-db.json。一个场景包含舞台参数、音频设置、循环设置、剧情流转配置、TTS 声线和图层数组；一个场景组则决定最终展示入口和组内逻辑关系。

场景字段

  • scene.focal、scene.parallax 控制透视焦距和头动幅度。
  • scene.audioAsset、audioLoop、gifLoop、webmLoop 控制媒体播放与循环。
  • scene.flow 决定场景结束后的跳转方式（none / auto / dialog）及路由关键词。
  • scene.xfyunVoice 指定本场景使用的讯飞 TTS 声线。
  • scene.ageRequired 设定场景对应的剧情节拍阶段。
  • scene.realtimeReply 控制是否开启实时流式回复。
  • items[] 保存每个图层的位置、深度（z）、缩放、旋转、倾斜和透明度。

场景组字段

  • sceneGroups 保存多个最终展示视频集合。
  • activeSceneGroupId 决定编辑器当前操作的组。
  • finalSceneGroupId 决定 final.html 默认展示哪个组。
  • finalStartSceneId 决定该组的首个播放场景。
当前 intro 页面会把“最终展示页”“手机端体验页”“编辑器”和 README 演示文档都作为作品入口呈现，因此场景组配置已经不只是编辑器内部数据，也直接影响面向观众的展示路线。

互动管线

三维视差渲染

摄像头或设备姿态 → 头部 / 视角状态 → 按图层深度计算偏移 → Canvas + 媒体叠层渲染

语音与剧情

语音或文字输入 → Web Speech / 文本 → Kimi 导演提示 → 讯飞 TTS → 角色语音、字幕、场景跳转

场景流转

当前场景播放 → 检测媒体结束 → none / auto / dialog → 停留或切换下一场景

运行部署

本地运行只需要启动 Node 服务。线上部署推荐把 Node 服务放在 PM2 后面，再由 Nginx 负责 HTTPS 和反向代理，因为摄像头与麦克风能力在现代浏览器中需要安全上下文。

本地入口

npm start

http://127.0.0.1:5174/intro.html
http://127.0.0.1:5174/index.html
http://127.0.0.1:5174/final.html

外部服务变量

  • PORT（默认 5174）
  • MOONSHOT_API_KEY、MOONSHOT_BASE_URL、KIMI_MODEL
  • KIMI_TIMEOUT_MS、KIMI_MAX_TOKENS
  • XFYUN_APP_ID、XFYUN_API_KEY、XFYUN_API_SECRET
  • XFYUN_TTS_URL、XFYUN_TTS_VOICE
